Introduction

The serverless framework is an application framework to manage your API code, cloud artifacts and provisioning tools at one place. It simply allows to define your microservices in code. The framework, uses plugin system to extend its core capabilities and provides the foundation to build web, mobile and IOT applications.

Architectural Considerations

When building a serverless application, several architectural considerations needs to be addressed. Most of them are common to micro-service architecture best practices.

  • Designing Microservices
  • Database Selection and Design
  • Dependency Management
  • Credentials Storage
  • DevOps
